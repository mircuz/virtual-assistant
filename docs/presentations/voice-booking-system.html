How It Actually Works

Voice Gateway (Fly.io)

Generates ephemeral OpenAI session tokens with tools & voice config. Proxies function calls from the browser to the booking API. Stateless — scales from zero.

Booking Engine (FastAPI)

9 REST endpoints: shops, staff, services, customers, availability, appointments. Pure business logic — no AI dependency. Runs on Lambda or any container.

OpenAI Realtime API

Browser connects directly via WebRTC. Handles all audio streaming, speech-to-text, LLM reasoning, and text-to-speech natively. Sub-second latency.

Databricks SQL Warehouse

All state lives in Delta tables — multi-tenant, ACID, queryable. Availability algorithm runs as SQL. Same warehouse powers analytics.

Build Your Own in an Afternoon

1. Define your tools

Write JSON schemas for what the AI can do. Each tool = one function call (check stock, create ticket, query a table — whatever your domain needs).

2. Build the API

FastAPI endpoint per tool. Read/write to Databricks SQL. The voice gateway proxies calls — your API doesn't know it's talking to a voice agent.

3. Configure the voice

Set the system prompt, personality, and voice. OpenAI handles all audio — STT, reasoning, TTS. You never touch audio bytes.

4. Deploy & iterate

Lambda for API, Fly.io for gateway. Both scale to zero. Test via browser WebRTC, then connect Twilio for real phone numbers.
